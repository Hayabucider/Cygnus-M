---

## 検証の定義

keymap を正として、Viewer が正しく反映されているかを確認する。
Viewer が合っていなければ Viewer を修正する。keymap を修正しない。

### 確認項目

| 確認項目 | keymap側 | Viewer側 |
|---|---|---|
| pos番号 | keymap 内の pos 定義 | ViewerのposMap定義 |
| キー数 | 各レイヤーのバインド数 | 表示キー数 |
| 親指キー順序 | 親指段の順 | Viewer の配置順 |
| エンコーダー | sensor-bindings の記述 | Viewer のencoder表示 |

## 2026-06-24（L8アクセス修正 + L8 JIS対応）

### 実施内容

1. **L8アクセス問題の解決**
   - 標準 `&lt 8 LANG2` では hold 判定が不安定でL8へ移動できなかった
   - `hold-preferred` フレーバーの専用 behavior `l8_lt` を `behaviors` ブロックに追加
   - L0/L2/L4/L6 の pos44 を `&lt 8 LANG2` → `&l8_lt 8 LANG2` に変更

2. **L8記号レイヤーのJIS対応**
   - `STAR` / `DQT` / `SQT` 等のUS配列前提コードを JIS 実キー操作に置き換え
   - `GRAVE` は JIS/Windows 環境で入力切替として動作するため `LS(LBKT)` で代替
   - L8 未使用キーを `&trans`（下位レイヤー透過）から `&none`（無効化）に変更

3. **Windows側キーボード設定変更**
   - 「英語キーボード 101/102」→「日本語キーボード 106/109」に変更

### 確認結果

| 項目 | 結果 | 備考 |
|---|---|---|
| pos番号 | ✅ 一致 | keymap コメントと viewer-spec.md は1-indexed で一致確認済み |
| キー数 | ✅ 51キー | 左26 + エンコーダー1（左手側）+ 右24 |
| 親指キー順序 | ✅ 一致 | pos44=英数/L8, pos45=かな/Alt, pos46=Space/Sub（左）/ pos47=→, pos48=BSPC, pos49=Enter/L9, pos50=DEL（右） |
| エンコーダー | ✅ 一致 | L8: PG_UP/PG_DN |
